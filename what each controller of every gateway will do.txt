what each controller of every gateway will do:
✅ 1) initiatePayment()

Create an order / transaction / request

Even ancient government portals do this in some form:

Generate transaction ID

Create redirect URL

Create session/token

Create hash/signature

Return form fields or URL to client

Every gateway ultimately has:

✅ Order creation
✅ Redirect

Even SBIEpay, eGRAS, PayGov, BillDesk, CCAvenue, Paytm, PhonePe — ALL of them.

✅ 2) verifyPayment()

Confirm payment status

Every gateway has one of these:

✅ callback response
✅ webhook
✅ status API
✅ checksum/hmac verification
✅ UTR/Bank Ref No

Even if the mechanism is totally different, the purpose is identical:

“Did the payment succeed or not?”

✅ 3) refundPayment()

Process refund or request reversal

Every gateway supports:

✅ full refund
✅ partial refund
✅ refund status
✅ settlement status

Even SBIEpay and ancient
